Preguntas

a. ¿Quién inventó el sistema de control de versión Git y por qué?

El sistema de control de versiones Git fue creado por Linus Torvalds en 2005. Linus Torvalds es conocido principalmente por ser el creador del núcleo de Linux. Git fue desarrollado por Torvalds como una solución para gestionar eficientemente el desarrollo colaborativo del kernel de Linux. Antes de Git, el desarrollo del kernel de Linux utilizaba otro sistema de control de versiones llamado BitKeeper. Sin embargo, en 2005, debido a ciertos desacuerdos sobre la licencia de BitKeeper, la comunidad de desarrollo de Linux tuvo que dejar de utilizarlo. Esto llevó a Torvalds a iniciar el desarrollo de una herramienta de control de versiones distribuido desde cero, lo que resultó en Git. Torvalds diseñó Git con el objetivo de tener un sistema rápido, escalable y altamente eficiente para el manejo de grandes proyectos de software con múltiples colaboradores. Git se ha vuelto extremadamente popular en la comunidad de desarrollo de software debido a su capacidad de gestionar fácilmente ramas (branches), fusiones (merges) y revisiones (commits), permitiendo a los desarrolladores trabajar de manera eficiente en paralelo y mantener un historial completo de los cambios realizados en el código fuente.

b. ¿A quién pertenece actualmente Github y por qué?

GitHub fue adquirido por Microsoft en junio de 2018. Microsoft adquirió GitHub por aproximadamente $7.5 mil millones de dólares. La adquisición se llevó a cabo con el objetivo de fortalecer la presencia de Microsoft en el ámbito del desarrollo de software y la computación en la nube, así como para expandir su compromiso con la comunidad de desarrolladores. Microsoft ha mantenido la plataforma GitHub como un servicio independiente y ha seguido invirtiendo en su desarrollo y expansión. La adquisición ha permitido a Microsoft aprovechar la gran base de usuarios y el ecosistema de desarrolladores de GitHub, además de integrar GitHub con sus propias herramientas y servicios, como Azure, Visual Studio y .NET, para proporcionar una experiencia de desarrollo más completa e integrada. En general, la adquisición de GitHub por parte de Microsoft ha sido bien recibida por la comunidad de desarrolladores, ya que ha brindado estabilidad y recursos adicionales para el crecimiento y la mejora continua de la plataforma.

c. ¿Hay otra forma que no sea la terminal para trabajar con Github?

Sí, además de utilizar la línea de comandos en la terminal, existen varias alternativas y herramientas gráficas que permiten trabajar con GitHub de manera más visual e intuitiva. Algunas de las opciones más populares son:

  1. GitHub Desktop:Es una aplicación oficial desarrollada por GitHub. Proporciona una interfaz gráfica de usuario (GUI) fácil de usar para administrar repositorios, realizar seguimiento de cambios, realizar confirmaciones (commits), fusionar ramas (branches) y realizar otras tareas comunes de control de versiones. Está disponible para Windows y macOS.
  2. Sourcetree:Es una herramienta gratuita y popular para Git y Mercurial. Proporciona una interfaz gráfica de usuario intuitiva que facilita la administración de repositorios, la visualización de historiales de cambios, la creación de ramas y la realización de confirmaciones. Está disponible para Windows y macOS.
  3. GitKraken:Es otra herramienta gráfica de control de versiones que admite Git. Ofrece una interfaz visual atractiva y potente para administrar repositorios, realizar fusiones, resolver conflictos, realizar confirmaciones y otras acciones. Está disponible para Windows, macOS y Linux.
Estas herramientas gráficas ofrecen una forma más amigable de interactuar con GitHub y facilitan la gestión del control de versiones, especialmente para aquellos que prefieren evitar el uso de la línea de comandos. Sin embargo, es importante destacar que tener conocimientos básicos de Git y la línea de comandos sigue siendo útil para comprender los conceptos subyacentes y tener un mayor control sobre el flujo de trabajo de desarrollo.
